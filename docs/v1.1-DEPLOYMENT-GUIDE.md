## 🚀 배포 단계별 가이드

### Step 1: Smart Contract 배포 (30분)

```bash
# 1.1 Foundry로 컴파일
forge build

# 1.2 Testnet 배포 (Polygon Mumbai)
forge create \
  --rpc-url https://rpc-mumbai.maticvigil.com \
  --private-key $PRIVATE_KEY \
  src/DXResolver-v1.1-Extended.sol:DXResolver \
  --constructor-args "0xYOUR_REGISTRY_ADDRESS"

# 출력 예:
# Deployed to: 0x1234567890123456789012345678901234567890

# 1.3 Contract 저장 (배포 후)
export RESOLVER_ADDRESS="0x1234567890123456789012345678901234567890"
```

### Step 2: Frontend 통합 (1시간)

```bash
# 2.1 의존성 설치
npm install \
  viem ethers unicode-normalization \
  lucide-react tailwindcss typescript

# 2.2 컴포넌트 통합
cp WalletAddressValidator-v1.1.js src/utils/
cp MultiLangTextInput-v1.1.jsx src/components/
cp ABIManager-v1.1.jsx src/components/

# 2.3 App.jsx에 import
import { MultiLangTextInput } from './components/MultiLangTextInput-v1.1';
import { ABIManager } from './components/ABIManager-v1.1';
import { validateAddress } from './utils/WalletAddressValidator-v1.1';

# 2.4 Environment 설정
cat > .env.local << 'DOTENV'
VITE_RESOLVER_ADDRESS=0x1234567890123456789012345678901234567890
VITE_REGISTRY_ADDRESS=0xYOUR_REGISTRY_ADDRESS
VITE_RPC_URL=https://polygon-rpc.com/
DOTENV
```

### Step 3: 테스트 (30분)

```bash
# 3.1 Smart Contract 테스트
forge test -vvv

# 3.2 Frontend 테스트
npm test

# 3.3 E2E 테스트 (수동)
- 한글 도메인 생성
- 중국어 설명 저장
- Bitcoin 지갑 추가
- ABI 저장 & 조회
- Homoglyph 감지 확인
```

### Step 4: Mainnet 배포 (30분)

```bash
# 4.1 Mainnet으로 배포 (비용 절감하려면 Polygon)
forge create \
  --rpc-url https://polygon-rpc.com \
  --private-key $PRIVATE_KEY \
  src/DXResolver-v1.1-Extended.sol:DXResolver \
  --constructor-args "0xYOUR_REGISTRY_ADDRESS" \
  --verify \
  --etherscan-api-key $POLYGONSCAN_API_KEY

# 4.2 배포 확인 (Polygonscan)
# https://polygonscan.com/address/YOUR_RESOLVER_ADDRESS
```

### Step 5: Frontend 배포 (30분)

```bash
# 5.1 Build
npm run build

# 5.2 Vercel 배포 (추천)
vercel deploy --prod

# 또는 traditional host
scp -r dist/ user@your-server:/var/www/dexignation/
```

## 🧪 테스트 시나리오

### Test 1: Multi-Wallet Support

```javascript
// Bitcoin 주소 검증
validateAddress(0, '1A1z7agoat7FYN82BWEimYWydLMQD6epP');
// ✅ { isValid: true, coin: "Bitcoin" }

// Solana 주소 검증
validateAddress(501, '9B5X4z8ThjnCVq2QV4QyDwQCa7L4JcMjkX7KP5sKS8N');
// ✅ { isValid: true, coin: "Solana" }

// Tron 주소 검증
validateAddress(195, 'TLAWdc8xk8PH8wiiX6pPXaeRUTAU6iTMzA');
// ✅ { isValid: true, coin: "Tron" }
```

### Test 2: Multi-Language Support

```javascript
// 한글 설명 저장
resolver.setMultiLangText(
  nodeHash,
  "description",
  "ko",
  "Web3 개발자"
);
// ✅ MultiLangTextChanged event emitted

// 중국어 설명 저장
resolver.setMultiLangText(
  nodeHash,
  "description",
  "zh",
  "Web3 开发者"
);
// ✅ MultiLangTextChanged event emitted

// 언어별 조회
resolver.getMultiLangText(nodeHash, "description", "ko");
// ✅ "Web3 개발자"

resolver.getMultiLangText(nodeHash, "description", "en");
// ✅ Falls back to English
```

### Test 3: Full ABI Support

```javascript
// Polygon에서 Uniswap ABI 저장
resolver.setABI(
  uniswap_node,
  137,  // Polygon
  4,    // JSON
  abiJson
);
// ✅ ABIChanged event emitted

// dApp에서 조회
(contentType, abiData) = resolver.ABI(uniswap_node, 137, 4);
// ✅ contentType: 4, abiData: JSON bytes
```

### Test 4: Homoglyph Detection

```javascript
// Frontend에서 Cyrillic 'А' 감지
detectHomoglyphs("Москва");
// ✅ [
//   { char: 'М', lang: 'Cyrillic', warning: 'Similar to Latin M' }
// ]

// 사용자에게 경고 표시
showWarning("Similar characters detected. Be careful with this name!");
```

## 🆘 Troubleshooting

### 가스 비용이 너무 높을 경우
→ Polygon 사용 (Ethereum 대신)  
→ Layer 2 최적화 실시

### 주소 검증 실패
→ Unicode normalization 확인  
→ Front-end validator와 on-chain validator 동기화

### ABI 저장 실패
→ JSON 형식 확인  
→ 가스 제한 증가

### 다국어 텍스트 저장 실패
→ 언어 코드 지원 확인 (supportedLanguages)  
→ Unicode NFC 정규화 확인
